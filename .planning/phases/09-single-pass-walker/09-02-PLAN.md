    **Step 1: Create ArrayBuffer** in `src/Gluey.Contract/ArrayBuffer.cs`:

    ```
    internal struct ArrayBuffer : IDisposable
    ```

    NOT readonly struct (needs mutable _count). Fields:
    - `private ParsedProperty[]? _entries` -- rented from ArrayPool
    - `private int _count` -- current number of stored elements
    - `private int _capacity`
    - `private Dictionary<int, (int Start, int Count)>? _regions` -- maps array ordinal to (startIndex, count) range in _entries

    Constructor: `internal ArrayBuffer(int initialCapacity = 16)` -- rents from ArrayPool, initializes _regions dictionary.

    `internal void Add(int arrayOrdinal, ParsedProperty element)`:
    - If at capacity, grow (rent new array 2x, copy, return old)
    - If _regions does not contain arrayOrdinal, add (Start=_count, Count=0)
    - Store element at _entries[_count], increment _count
    - Update region: increment Count

    **Important:** Elements for the same arrayOrdinal must be added consecutively (walker processes array elements in order). The region tracks contiguous ranges.

    `internal ParsedProperty Get(int arrayOrdinal, int elementIndex)`:
    - If _regions is null or does not contain arrayOrdinal, return Empty
    - Get (Start, Count) from _regions
    - If elementIndex < 0 or >= Count, return Empty
    - Return _entries[Start + elementIndex]

    `internal int GetCount(int arrayOrdinal)`:
    - Return region Count, or 0 if not found

    `public void Dispose()`:
    - Return _entries to ArrayPool if not null

    **Step 2: Update WalkObject to create ParsedProperty with child references:**

    When storing a ParsedProperty for an object-typed property in the OffsetTable, use the new ParsedProperty constructor that includes childTable (_table) and childOrdinals. The child ordinals for a specific property are the subset of _nameToOrdinal that corresponds to that property's children. **Key insight:** _nameToOrdinal already contains full paths like "/address/street" -> ordinal. For a property at path "/address", its child ordinals need a local mapping: "street" -> ordinal of "/address/street". Build this mapping during WalkObject by collecting {localName -> ordinal} for each child property encountered.

    Concretely, in WalkObject:
    - Create a `Dictionary<string, int>? childOrdinals = null` at the start
    - For each property processed, if _buildOffsets and _nameToOrdinal has an ordinal for this child's full path, add to childOrdinals: `childOrdinals[localName] = ordinal`
    - When creating the ParsedProperty for the parent object node (the one stored at the parent's ordinal), pass childOrdinals and _table so its string indexer can resolve children

    **Wait -- the parent object's ParsedProperty is stored at the parent ordinal.** The parent's byte range covers the entire object (StartObject to EndObject). The parent ParsedProperty needs: buffer, offset (of StartObject), length (EndObject - StartObject), path, plus childTable=_table, childOrdinals={child local names -> ordinals}.

    For the parent object's byte range: record the reader's ByteOffset at StartObject, then after EndObject compute length. Store parent ParsedProperty in OffsetTable at parent's ordinal.
