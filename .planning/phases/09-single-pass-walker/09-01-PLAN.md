    Internal ref struct with fields: `_reader` (JsonByteReader), `_root` (SchemaNode), `_data` (byte[] -- null for Span overload), `_errors` (ErrorCollector), `_table` (OffsetTable), `_nameToOrdinal` (Dictionary), `_buildOffsets` (bool), `_assertFormat` (bool).

    Static entry points:
    - `internal static WalkResult Walk(byte[] data, SchemaNode root, Dictionary<string, int> nameToOrdinal, int propertyCount, bool assertFormat)` -- creates reader from data, OffsetTable(propertyCount), ErrorCollector, calls WalkValue on root
    - `internal static WalkResult Walk(ReadOnlySpan<byte> data, SchemaNode root, int propertyCount, bool assertFormat)` -- same but _buildOffsets=false, _data=null, no OffsetTable allocation (use default)

    **Core methods:**

    `private bool WalkValue(SchemaNode node)`:
    1. Follow $ref: `var effective = node.ResolvedRef ?? node;`
    2. Boolean schema check: if false, call SkipValue() and add error; if true, call SkipValue() and return true
    3. Read next token via `_reader.Read()`. If read fails, set structural error flag and return false
    4. Determine token type. For StartObject -> call WalkObject(effective). For StartArray -> call WalkArray(effective). For scalar -> validate inline.
    5. Scalar validation ordering (per locked decision):
       a. type keyword (if fails and type is set, skip type-dependent keywords)
       b. enum / const
       c. Type-dependent: numeric constraints for Number tokens, string constraints for String tokens
       d. Format (if _assertFormat and effective.Format is not null)
       e. Store in OffsetTable if _buildOffsets and this property has an ordinal
    6. Composition: allOf/anyOf/oneOf/not -- for scalar values, run each subschema's validators against the same token data. Use a temporary ErrorCollector per subschema to count pass/fail. NOTE: for composition, create helper `private bool ValidateScalarAgainstSchema(SchemaNode subNode, JsonByteTokenType tokenType, bool isInteger, ReadOnlySpan<byte> valueBytes, string path)` that runs type/enum/const/numeric/string/format validators without re-reading tokens.
    7. Conditionals: if/then/else using same captured token approach
    8. Return true if no errors for this node (local evaluation)

    `private bool WalkObject(SchemaNode node)`:
    1. HashSet<string> seenProperties for tracking
    2. int propertyCount = 0
    3. Loop: Read PropertyName token. If EndObject, break.
    4. Get property name from reader bytes (Encoding.UTF8.GetString for property name -- this is a necessary allocation, property names are used for dictionary lookups)
    5. seenProperties.Add(name), propertyCount++
    6. Find child schema: node.Properties?.TryGetValue(name, out childSchema)
    7. If no child schema: check additionalProperties (null=allow, BooleanSchema false=reject via ValidateAdditionalProperty). Also check patternProperties for matching patterns.
    8. If childSchema found or patternProperties match: WalkValue(childSchema) recursively. Before the recursive call, note the reader position for OffsetTable storage.
    9. If _buildOffsets and _nameToOrdinal.TryGetValue for the property name gives an ordinal, create ParsedProperty from _data + reader ByteOffset/ByteLength and call _table.Set(ordinal, property).
    10. After loop: ValidateRequired, ValidateMinProperties, ValidateMaxProperties
    11. DependentRequired: for each trigger in node.DependentRequired, if seenProperties contains trigger, validate
    12. DependentSchemas: for each trigger in node.DependentSchemas, if seenProperties contains trigger, run subschema validators against captured object state
    13. PropertyNames: for each seen property name, validate against node.PropertyNames schema
    14. Composition/conditionals at the object level (allOf etc. that apply to the object itself -- run structural validators like required/minProperties against captured state for each subschema)

    `private bool WalkArray(SchemaNode node)`:
    1. int elementCount = 0, int containsMatchCount = 0
    2. Lists for uniqueItems: byte[][] elementBytes, bool[] isNumber (allocate if node.UniqueItems == true)
    3. Loop: Read next token. If EndArray, break.
    4. Get item schema via KeywordValidator.GetItemSchema(node.PrefixItems, node.Items, elementCount)
    5. Recursively WalkValue(itemSchema ?? SchemaNode.True) -- need to "unread" token... Actually, since WalkValue reads the first token, the array loop should peek or the design should be: WalkValue assumes the next Read() gives the value token. So in WalkArray, after detecting it's not EndArray, we need to handle the fact that we already consumed the token. **Important design note:** WalkValue should NOT call Read() for the first token when called from WalkArray/WalkObject. Instead, WalkValue receives the already-read token type and processes from there. OR: WalkArray peeks at the next token type (calls Read), checks if EndArray, if not EndArray passes context to WalkValue which handles the already-read token. The cleanest approach: WalkValue always expects the current token to already be read (the caller reads it). WalkObject reads PropertyName then calls WalkValue which reads the value. WalkArray reads the element token, checks EndArray, if not calls a method that validates the current token.

    **Revised approach for token reading:**
    - The top-level Walk() calls `_reader.Read()` then `WalkValue()` which processes the current token (already read)
    - WalkObject: after reading PropertyName, calls `_reader.Read()` then WalkValue() for the value
    - WalkArray: calls `_reader.Read()`, if EndArray break, else WalkValue() processes current token
    - WalkValue for StartObject/StartArray delegates to WalkObject/WalkArray which handle their own internal reading

    6. If node.Contains is not null, validate element against Contains schema (use temp ErrorCollector), increment containsMatchCount if passes
    7. If uniqueItems, capture element bytes (slice from _data or reader span)
    8. elementCount++
    9. After loop: ValidateMinItems, ValidateMaxItems, ValidateContains (matchCount, minContains, maxContains), ValidateUniqueItems

    `private void SkipValue()`:
    Read and skip a complete JSON value (handles nested objects/arrays by tracking depth). Used for boolean schema true/false cases and for properties with no schema interest.
