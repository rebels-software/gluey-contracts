    - Required property with special chars (~ and /) => error path has RFC 6901 escaping via BuildChildPath
    - Empty required array => true (vacuously true)
    - Nested path: parent="/address", missing="street" => error path="/address/street"

    ValidateAdditionalProperty tests:
    - Property in Properties dictionary => true (always allowed)
    - Property NOT in Properties, additionalProperties=null => true (spec default allows)
    - Property NOT in Properties, additionalProperties=SchemaNode.False => false, AdditionalPropertyNotAllowed error
    - Property NOT in Properties, additionalProperties=SchemaNode.True => true (explicit allow-all)
    - Property NOT in Properties, additionalProperties=some schema node => true (schema validation deferred to walker)
    - Properties is null, additionalProperties=null => true (no constraints)
    - Properties is null, additionalProperties=SchemaNode.False => false (no known properties, all are additional)
  </behavior>

    GREEN phase: Add to KeywordValidator.cs (which already exists from plan 01):

    1. ValidateRequired(string[] required, HashSet<string> seenProperties, string path, ErrorCollector collector) -> bool
       - Iterate required array
       - For each name not in seenProperties: collector.Add with RequiredMissing error
       - Path for error: SchemaNode.BuildChildPath(path, required[i])
       - Track valid = true, set to false on any miss
       - Return valid (NOT early-return -- collect all missing property errors)

    2. ValidateAdditionalProperty(string propertyName, Dictionary<string, SchemaNode>? properties, SchemaNode? additionalProperties, string path, ErrorCollector collector) -> bool
       - If properties is not null AND properties.ContainsKey(propertyName): return true
       - If additionalProperties is null: return true (spec default)
       - If additionalProperties.BooleanSchema == false: collector.Add with AdditionalPropertyNotAllowed, return false
       - Else: return true (schema-based additionalProperties validated by walker in Phase 9)
       - Error path: SchemaNode.BuildChildPath(path, propertyName)
