    GREEN phase: Implement KeywordValidator.cs in src/Gluey.Contract.Json/:

    1. MapTokenToSchemaType(JsonByteTokenType tokenType, bool isInteger) -> SchemaType
       - Switch on tokenType: Null->Null, True/False->Boolean, String->String, StartObject->Object, StartArray->Array
       - Number: if isInteger => SchemaType.Integer | SchemaType.Number, else => SchemaType.Number
       - Default => SchemaType.None

    2. IsInteger(ReadOnlySpan<byte> numberBytes) -> bool
       - Create Utf8JsonReader over the span, Read(), then TryGetInt64(out _)
       - Return true only if both Read succeeds with Number token AND TryGetInt64 succeeds

    3. ValidateType(SchemaType expected, JsonByteTokenType tokenType, bool isInteger, string path, ErrorCollector collector) -> bool
       - Map token to SchemaType via MapTokenToSchemaType
       - If (expected & actual) != 0, return true
       - Else: collector.Add with TypeMismatch error, return false

    4. ValidateEnum(byte[][] enumValues, ReadOnlySpan<byte> tokenBytes, bool tokenIsNumber, string path, ErrorCollector collector) -> bool
       - Iterate enumValues, check tokenBytes.SequenceEqual(entry) for each
       - If no byte match AND tokenIsNumber, iterate again trying TryNumericEqual
       - If still no match: collector.Add with EnumMismatch, return false

    5. ValidateConst(byte[] expected, ReadOnlySpan<byte> tokenBytes, bool tokenIsNumber, string path, ErrorCollector collector) -> bool
       - If tokenBytes.SequenceEqual(expected), return true
       - If tokenIsNumber && TryNumericEqual(tokenBytes, expected, out bool equal) && equal, return true
       - Else: collector.Add with ConstMismatch, return false

    6. TryNumericEqual(ReadOnlySpan<byte> a, ReadOnlySpan<byte> b, out bool equal) -> bool (private)
       - Create Utf8JsonReader for each span, Read(), TryGetDecimal() on both
       - If both succeed: equal = (valA == valB), return true
       - Else: equal = false, return false

    REFACTOR: Extract common patterns if any duplication emerges.

    All methods use ValidationErrorMessages.Get() for error message strings. Never check ErrorCollector.IsFull — just call Add().
