  <files>

  <behavior>
    - ValidateUniqueItems with empty array returns true
    - ValidateUniqueItems with single element returns true
    - ValidateUniqueItems with [1, 2, 3] (distinct bytes) returns true
    - ValidateUniqueItems with [1, 1] (duplicate bytes) returns false with UniqueItemsViolation
    - ValidateUniqueItems with [1, 1.0] (numeric equivalence) returns false with UniqueItemsViolation
    - ValidateUniqueItems with [1e2, 100] (numeric equivalence) returns false with UniqueItemsViolation
    - ValidateUniqueItems with ["a", "b", "a"] returns false (string duplicates)
    - ValidateUniqueItems with [true, false, true] returns false (boolean duplicates)
    - ValidateUniqueItems with [null, null] returns false
    - ValidateUniqueItems with [1, "1"] returns true (different types, not duplicates)
    - Hashes use stackalloc for arrays <= 128 items (verified by code review, not test)
  </behavior>
  <action>
1. In KeywordValidator.cs, change `TryNumericEqual` from `private` to `internal` so ArrayValidator can reuse it.

2. Add to ArrayValidator:
   - `internal static bool ValidateUniqueItems(ReadOnlySpan<byte>[] elementBytes, bool[] isNumber, string path, ErrorCollector collector)`
   - Early return true for count <= 1
   - Compute FNV-1a hash for each element into `stackalloc int[count]` when count <= 128, else `new int[count]` fallback
   - O(n^2) comparison loop: skip if hashes differ, then SequenceEqual for byte-exact match, then TryNumericEqual fallback for number pairs
   - On first duplicate found: add UniqueItemsViolation error and return false
   - Private helper: `static int Fnv1aHash(ReadOnlySpan<byte> bytes)` using standard FNV-1a (offset basis 2166136261, prime 16777619, unchecked)

3. Write tests in UniqueItemsValidatorTests.cs. For each test, construct ReadOnlySpan<byte>[] from UTF-8 encoded JSON values and bool[] isNumber flags. Test all behavior items above. Use `System.Text.Encoding.UTF8.GetBytes()` to create byte arrays from JSON value strings.
