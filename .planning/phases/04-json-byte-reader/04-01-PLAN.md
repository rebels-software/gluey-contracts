Create `src/Gluey.Contract.Json/JsonByteReader.cs`:

`internal ref struct JsonByteReader` in namespace `Gluey.Contract.Json`:

**Fields:**
- `private Utf8JsonReader _reader` — the wrapped BCL reader
- `private JsonReadError _error` — populated on structural error

**Constructor:**
- `public JsonByteReader(ReadOnlySpan<byte> utf8Json)` — creates Utf8JsonReader with `JsonReaderOptions { CommentHandling = JsonCommentHandling.Skip, AllowTrailingCommas = true }`
- byte[] and ReadOnlyMemory<byte> implicitly convert to ReadOnlySpan<byte> — no overloads needed (but add a ReadOnlyMemory<byte> constructor that calls `.Span` for API clarity if desired)

**Properties:**
- `public JsonByteTokenType TokenType { get; private set; }`
- `public int ByteOffset { get; private set; }`
- `public int ByteLength { get; private set; }`
- `public JsonReadError Error => _error`
- `public bool HasError => _error.Kind != JsonReadErrorKind.None`

**Methods:**
- `public bool Read()`:
  1. Try `_reader.Read()` — if returns false, return false (end of data)
  2. Map `_reader.TokenType` to `JsonByteTokenType` via private `MapTokenType()` method
  3. Compute offsets via private `ComputeOffsets()`:
     - String/PropertyName: `ByteOffset = (int)_reader.TokenStartIndex + 1`, `ByteLength = _reader.ValueSpan.Length`
     - All others: `ByteOffset = (int)_reader.TokenStartIndex`, `ByteLength = _reader.ValueSpan.Length`
  4. Catch `JsonException`:
     - Determine error kind: check if `_reader.BytesConsumed >= utf8Json.Length` for UnexpectedEndOfData, else InvalidJson
     - Create `JsonReadError` with kind, `(int)_reader.BytesConsumed` as byte offset, `ex.Message`
     - Return false

- `private static JsonByteTokenType MapTokenType(System.Text.Json.JsonTokenType tokenType)` — switch mapping BCL enum to our enum. Map `JsonTokenType.Comment` to `JsonByteTokenType.None` (comments are skipped). Map `JsonTokenType.None` to `JsonByteTokenType.None`.

**Critical offset note:** For structural tokens ({, }, [, ]), `_reader.ValueSpan.Length` gives the length of the delimiter (1 byte). This is correct — use it directly. However, for `JsonTokenType.None` (before first Read), ValueSpan is empty — handle by not updating offsets until first successful Read().

**Important:** The reader needs to store the input length to detect truncation. Store `private readonly int _inputLength` from the span length in the constructor. In the catch block, compare `_reader.BytesConsumed` against `_inputLength` to choose between UnexpectedEndOfData and InvalidJson.
