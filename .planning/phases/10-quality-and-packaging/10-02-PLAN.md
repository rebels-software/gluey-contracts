Create the Scenarios/ and Payloads/ directories under benchmarks/Gluey.Contract.Benchmarks/.

**PayloadGenerator.cs** -- Static class with methods to generate byte[] payloads at three sizes:
- `GenerateFlat(int targetBytes)` -- flat JSON object with string/int/bool properties, repeated to fill target size
- `GenerateNested(int targetBytes)` -- nested objects 2-3 levels deep (address.street, address.city, etc.)
- `GenerateArray(int targetBytes)` -- object with an array of items, each item a small object
- `GenerateFullSchema(int targetBytes)` -- complex payload exercising composition, conditionals, patterns

Each returns `byte[]`. Use `System.Text.Encoding.UTF8.GetBytes()` on constructed JSON strings. Payloads must be valid JSON matching their respective schemas.

**Each benchmark class** follows this pattern (per user decision):
- `[MemoryDiagnoser]` attribute on the class
- `[SimpleJob(RuntimeMoniker.Net90)]` attribute
- `GlobalSetup` loads schema via `JsonContractSchema.Load(schemaString)` and generates three payloads (small ~100B, medium ~5KB, large ~50KB)
- Six TryParse benchmarks: `TryParse_Small`, `TryParse_Medium`, `TryParse_Large` (byte[] overload)
- Six ValidateOnly benchmarks: `ValidateOnly_Small`, `ValidateOnly_Medium`, `ValidateOnly_Large` (ReadOnlySpan<byte> overload)
- Three `StjDeserialize_*` baselines marked `[Benchmark(Baseline = true)]` using `JsonDocument.Parse()` for comparison
- Each benchmark method calls TryParse, then Dispose() on the result, returning the bool

**Scenario schemas:**

1. **FlatObjectBenchmark**: Schema with `type: object`, 5-10 properties (name:string, age:integer, active:boolean, score:number, email:string with format). Properties have `required`.

2. **NestedObjectBenchmark**: Schema with nested `properties` -- top-level has `address` object with `street`, `city`, `zip`; `contact` object with `phone`, `email`. 2-3 levels deep.

3. **ArrayPayloadBenchmark**: Schema with `items` defining array element schema. Array of objects with 3-4 properties each. Tests array-heavy payloads. Note: ArrayBuffer allocates for array scenarios -- this is expected and documented.

4. **FullSchemaBenchmark**: Schema using allOf/anyOf, if/then/else, pattern, required, minLength/maxLength, minimum/maximum. Exercises the full validation pipeline.

Use `Gluey.Contract.Json` namespace imports. The schemas should be loaded via `JsonContractSchema.Load(schemaJsonString)`.

**Allocation test pattern** (all files use this):
```csharp
using FluentAssertions;
using Gluey.Contract.Json;

namespace Gluey.Contract.Json.Tests.AllocationTests;
```

Each test class has a `[SetUp]` or `[OneTimeSetUp]` that loads a schema and prepares a payload as `byte[]`. Tests use non-array schemas (to avoid ArrayBuffer class allocation) except where testing array scenarios specifically.

**Helper method** in each class (or a shared base):
```csharp
private static long MeasureAllocations(Action action)
{
    action(); // warm up (JIT)
    long before = GC.GetAllocatedBytesForCurrentThread();
    action();
    long after = GC.GetAllocatedBytesForCurrentThread();
    return after - before;
}
```

**TryParseAllocationTests.cs** -- Coverage area 1 and 2:
- `TryParse_ByteArray_AllocatesZeroBytes`: Full TryParse(byte[]) path with a flat-object schema, assert 0 bytes
- `TryParse_ReadOnlySpan_AllocatesZeroBytes`: Validate-only TryParse(ReadOnlySpan<byte>) path, assert 0 bytes
- Use a simple schema: `{"type":"object","properties":{"name":{"type":"string"},"age":{"type":"integer"}},"required":["name","age"]}`
- Use a matching payload: `{"name":"test","age":42}`

**PropertyAccessAllocationTests.cs** -- Coverage area 3:
- `PropertyAccess_StringIndexer_AllocatesZeroBytes`: After TryParse, access `result["name"]` and `result["age"]`, assert 0 bytes for the indexer access itself (NOT for GetString() which materializes a string)
- `PropertyAccess_OrdinalIndexer_AllocatesZeroBytes`: Access `result[0]`, `result[1]`, assert 0 bytes
- Note: GetString() allocates a string -- that's expected. Only test the indexer lookup, not value materialization.

**DisposeAllocationTests.cs** -- Coverage area 4:
- `Dispose_AllocatesZeroBytes`: After TryParse, call Dispose(), assert 0 bytes
- `DoubleDispose_AllocatesZeroBytes`: Call Dispose() twice, assert 0 bytes (verify idempotent dispose works)

**FormatAssertionAllocationTests.cs** -- Separate budget:
- `TryParse_WithFormatAssertion_AllocationBudget`: TryParse with SchemaOptions.AssertFormat = true, schema with format keywords. This is ALLOWED to allocate. Test asserts the allocation is reasonable (e.g., < 1000 bytes) rather than zero. Documents the known allocation from format assertion as opt-in.

All tests use NUnit `[Test]` attribute and FluentAssertions.
