**ParseResult refactor** -- Eliminate the `new int[1]` allocation in ParseResult constructors. The double-dispose guard must become zero-allocation. Strategy: make double-dispose benign without a guard. Remove the `_disposedHolder` field entirely. In `Dispose()`, simply call `_offsetTable.Dispose()`, `_errorCollector.Dispose()`, and `_arrayBuffer?.Dispose()` unconditionally. These underlying Dispose methods already use ArrayPool.Return which is safe to call with already-returned buffers (returns are idempotent when the buffer has been cleared from tracking). If OffsetTable or ErrorCollector Dispose is not idempotent, make it so by adding a `_disposed` bool field to those structs (they are already mutable internally via array references). Check OffsetTable.Dispose() and ErrorCollector.Dispose() -- if they use ArrayPool<T>.Shared.Return(), calling Return twice with the same buffer is technically safe (it just returns it again), but to be clean, null out the internal array reference after return so subsequent Dispose is a no-op.

Update both ParseResult constructors to remove `_disposedHolder = new int[1];` and remove the `_disposedHolder` field declaration.

Update `Dispose()` to:
```csharp
public void Dispose()
{
    _offsetTable.Dispose();
    _errorCollector.Dispose();
    _arrayBuffer?.Dispose();
}
```

**Verify OffsetTable and ErrorCollector idempotent dispose:** Read their Dispose implementations. If they already null-check before ArrayPool.Return, no changes needed. If not, add null-check-and-clear pattern to make them idempotent (e.g., `var arr = _buffer; _buffer = null; if (arr != null) ArrayPool.Return(arr);` -- note: readonly struct fields can't be reassigned, so check if they use a holder pattern like ErrorCollector's int[1] count holder. If the array field itself is a reference type field in a readonly struct, it CAN be reassigned because it's the reference that's readonly, not the array contents. Actually in readonly structs, ALL fields including reference-type fields are readonly and cannot be reassigned. So use Interlocked.Exchange on the array field to atomically swap to null.)
