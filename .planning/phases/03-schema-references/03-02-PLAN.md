Create `src/Gluey.Contract.Json/SchemaRefResolver.cs`:
- `internal static class SchemaRefResolver` in `Gluey.Contract.Json` namespace
- Main entry: `internal static bool TryResolve(SchemaNode root, SchemaRegistry? registry)`
- Two-pass algorithm per CONTEXT.md and RESEARCH.md:

**Pass 1 -- Collect anchors:**
- Walk entire tree recursively
- For each node with non-null Anchor, add to `Dictionary<string, SchemaNode>` keyed by anchor name
- If duplicate anchor name found, return false (AnchorDuplicate error)
- Also track $id nodes for potential base URI scoping (basic support -- track the mapping but keep simple)

**Pass 2 -- Resolve refs:**
- Walk entire tree recursively
- For each node with non-null Ref:
  1. Classify the $ref value:
     - Starts with "#/" -> JSON Pointer (resolve within current schema)
     - Starts with "#" but not "#/" -> Anchor reference (look up in anchor map)
     - "#" alone -> root of current schema
     - Contains "://" or no "#" prefix -> Cross-schema URI (look up in registry)
     - Has URI + fragment -> Split at "#", look up base in registry, then resolve fragment within that schema's root
  2. Resolve to target SchemaNode
  3. Cycle detection: use per-resolution-chain `HashSet<string>` tracking node.Path. Add target path before recursing into target's own $ref chain. Remove after. If target.Path already in set, cycle detected -> return false.
  4. Set node.ResolvedRef = target

**JSON Pointer navigation** (private helper):
- Split pointer (after "#/") by "/"
- Unescape each segment: replace "~1" with "/" first, then "~0" with "~" (order matters per RFC 6901)
- Navigate: "$defs" -> Defs dict, "properties" -> Properties dict, "items" -> Items, "allOf"/"anyOf"/"oneOf" -> array index, "prefixItems" -> array index, "additionalProperties" -> AdditionalProperties, "if"/"then"/"else"/"not" -> corresponding property, "contains" -> Contains, "patternProperties" -> PatternProperties dict, "dependentSchemas" -> DependentSchemas dict, "propertyNames" -> PropertyNames
- Return null if any segment fails to navigate

**Tree walk helper** (for both passes): Must visit ALL children -- Properties values, Defs values, Items, PrefixItems elements, AllOf/AnyOf/OneOf elements, Not, If/Then/Else, AdditionalProperties, Contains, PatternProperties values, DependentSchemas values, PropertyNames. Same traversal pattern as SchemaIndexer.WalkNode.
