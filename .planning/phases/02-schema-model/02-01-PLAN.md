```csharp
namespace Gluey.Contract;

/// <summary>
/// Represents the JSON Schema "type" keyword values as a flags enum.
/// Multiple types can be combined (e.g., <c>SchemaType.String | SchemaType.Null</c>
/// for <c>"type": ["string", "null"]</c>).
/// </summary>
[Flags]
internal enum SchemaType : byte
{
    None    = 0,
    Null    = 1 << 0,   // 1
    Boolean = 1 << 1,   // 2
    Integer = 1 << 2,   // 4
    Number  = 1 << 3,   // 8
    String  = 1 << 4,   // 16
    Array   = 1 << 5,   // 32
    Object  = 1 << 6,   // 64
}
```

This is a byte-sized enum supporting bitwise OR for multi-type schemas like `"type": ["string", "null"]`. Validation (Phase 5+) will use bitwise AND for O(1) type matching.

**Static helper method for path building:**
```csharp
internal static string BuildChildPath(string parentPath, string propertyName)
{
    // RFC 6901: encode ~ as ~0, / as ~1 (order matters!)
    string escaped = propertyName.Replace("~", "~0").Replace("/", "~1");
    return parentPath.Length == 0
        ? "/" + escaped
        : parentPath + "/" + escaped;
}
```
Root path is `""` (empty string per RFC 6901). Child paths: `"/propertyName"`. Nested: `"/address/street"`.

**Important design notes:**
- enum/const stored as raw UTF-8 byte arrays (not JsonElement) to avoid JsonDocument lifetime management. Each enum value and const are serialized to `byte[]` by the loader. Byte-level comparison in Phase 5 can compare directly.
- SchemaNode is a class (not struct) because tree nodes reference children. Allocated once at load time, not on the parse path.
- All properties get-only -- immutability enforced by compiler.
