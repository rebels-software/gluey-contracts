**VALD-01: Numeric min/max validation (at least 4 tests):**

Test 1 - `Parse_NumericBelowMin_CollectsMinimumExceededError`:
Contract: `temperature` field as `int16` size 2 with `"validation": { "min": -40, "max": 85 }`, followed by `humidity` as `uint8` size 1 depending on temperature.
Payload: temperature = -50 (0xCE, 0xFF little-endian), humidity = 50 (0x32).
Assert: `result.IsValid` is false, `result.Errors.Count` is 1, `result.Errors[0].Path` is `"/temperature"`, `result.Errors[0].Code` is `ValidationErrorCode.MinimumExceeded`. Value still accessible: `result["temperature"].GetInt16()` returns -50.

Test 2 - `Parse_NumericAboveMax_CollectsMaximumExceededError`:
Same contract. Payload: temperature = 100 (0x64, 0x00), humidity = 50.
Assert: error code is `MaximumExceeded`, path is `"/temperature"`. Value accessible: `GetInt16()` returns 100.

Test 3 - `Parse_NumericWithinRange_NoErrors`:
Same contract. Payload: temperature = 25 (0x19, 0x00), humidity = 50.
Assert: `result.IsValid` is true, `result.Errors.Count` is 0.

Test 4 - `Parse_NumericAtBoundary_NoErrors`:
Same contract. Payload: temperature = 85 (0x55, 0x00) -- exactly at max. humidity = 50.
Assert: `result.IsValid` is true (boundary is inclusive, `value > max` is the check).

**VALD-02: String pattern validation (at least 2 tests):**

Test 5 - `Parse_StringMatchesPattern_NoErrors`:
Contract: `deviceId` as `string` size 8 encoding `ASCII` with `"validation": { "pattern": "^DEV-[0-9]{4}$" }`.
Payload: ASCII bytes for "DEV-1234" (8 bytes: 0x44, 0x45, 0x56, 0x2D, 0x31, 0x32, 0x33, 0x34).
Assert: `result.IsValid` is true.

Test 6 - `Parse_StringFailsPattern_CollectsPatternMismatchError`:
Same contract. Payload: "INVALID!" (8 ASCII bytes: 0x49, 0x4E, 0x56, 0x41, 0x4C, 0x49, 0x44, 0x21).
Assert: `result.Errors[0].Code` is `PatternMismatch`, path is `"/deviceId"`. Value accessible: `GetString()` returns `"INVALID!"`.

**VALD-03: String minLength/maxLength validation (at least 3 tests):**

Test 7 - `Parse_StringTooShort_CollectsMinLengthExceededError`:
Contract: `label` as `string` size 10 encoding `ASCII` with `"validation": { "minLength": 3, "maxLength": 8 }`.
Payload: "AB" followed by 8 null bytes (0x41, 0x42, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00).
Assert: error code is `MinLengthExceeded`, path is `"/label"`. After trimming nulls, `GetString()` returns `"AB"` (length 2 < minLength 3).

Test 8 - `Parse_StringTooLong_CollectsMaxLengthExceededError`:
Same contract. Payload: "ABCDEFGHI\0" (9 meaningful chars + 1 null in 10 bytes: 0x41-0x49, 0x00).
Assert: error code is `MaxLengthExceeded`. `GetString()` returns `"ABCDEFGHI"` (length 9 > maxLength 8).

Test 9 - `Parse_StringWithinLengthRange_NoErrors`:
Same contract. Payload: "HELLO\0\0\0\0\0" (5 chars + 5 nulls).
Assert: `result.IsValid` is true. `GetString()` returns `"HELLO"` (length 5, within 3-8 range).

**VALD-04: Payload too short returns null (at least 2 tests):**

Test 10 - `Parse_PayloadTooShort_ReturnsNull`:
Contract: two fields totaling 5 bytes (`uint16` size 2 + `uint16` size 2 + `uint8` size 1).
Payload: only 3 bytes (0x01, 0x02, 0x03).
Assert: `schema.Parse(payload)` returns `null`.

Test 11 - `Parse_PayloadExactSize_ReturnsResult`:
Same contract. Payload: exactly 5 bytes.
Assert: `schema.Parse(payload)` is not null.

**VALD-05: Multiple errors collected (at least 2 tests):**

Test 12 - `Parse_MultipleFieldsInvalid_CollectsAllErrors`:
Contract: `temperature` int16 size 2 with min -40 max 85, `humidity` uint8 size 1 depending on temperature with min 0 max 100, `pressure` uint16 size 2 depending on humidity with min 300 max 1100.
Payload: temperature = 100 (above max), humidity = 150 (above max), pressure = 200 (below min).
Assert: `result.Errors.Count` is 3. Check that each error has the correct path (`"/temperature"`, `"/humidity"`, `"/pressure"`) and code. All three values are still accessible via their respective `GetXxx()` methods.

Test 13 - `Parse_MixedNumericAndStringErrors_CollectsAll`:
Contract: `temperature` int16 size 2 with max 85, followed by `deviceId` string size 8 encoding ASCII with pattern `"^DEV-"` depending on temperature.
Payload: temperature = 100, deviceId = "INVALID!" (8 bytes).
Assert: `result.Errors.Count` is 2. One `MaximumExceeded` for temperature, one `PatternMismatch` for deviceId.

**Additional edge case tests (at least 2 more):**

Test 14 - `Parse_UnsignedFieldWithMinMax_Validates`:
Contract: `sensorId` uint32 size 4 with min 1000 max 9999.
Payload: sensorId = 500 (below min).
Assert: `MinimumExceeded` error.

Test 15 - `Parse_FloatFieldWithMinMax_Validates`:
Contract: `voltage` float32 size 4 with min 0.0 max 5.0.
Payload: voltage = 6.5 (above max). Use `BitConverter.GetBytes(6.5f)` for little-endian bytes.
Assert: `MaximumExceeded` error. Value accessible: `GetFloat32()` returns approximately 6.5.

**Byte construction notes:**
- Use `BitConverter.GetBytes()` or `BinaryPrimitives.WriteXxxLittleEndian()` to build payloads
- For int16 little-endian: -50 = 0xCE, 0xFF; 100 = 0x64, 0x00; 25 = 0x19, 0x00; 85 = 0x55, 0x00
- For ASCII strings: use `Encoding.ASCII.GetBytes()` or literal hex bytes
- All contracts use `"endianness": "little"` for test simplicity
