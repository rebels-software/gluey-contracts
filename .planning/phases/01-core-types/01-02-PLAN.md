Create src/Gluey.Contract/ErrorCollector.cs:

1. public readonly struct ErrorCollector : IDisposable
2. Fields: ValidationError[] _errors (rented from ArrayPool), int _capacity
3. Need a mutable count -- since readonly struct fields cannot be mutated, use a small int[] of length 1 rented alongside, OR use an internal wrapper. Simplest approach: store count in a single-element int[] (cheap, avoids boxing). Alternative: since ErrorCollector is always used internally and passed by reference, the _errors array contents are mutable even in readonly struct. Track count as _errors array usage.

   RECOMMENDED approach: Use a mutable count field pattern. Since ErrorCollector is a readonly struct, its fields cannot be reassigned. However, we need a mutable count. Options:
   - Option A: Make _count a field and accept that ErrorCollector must be used via `ref` or as a local variable (not passed by value after construction). Since it is always used internally within the parse pipeline, this is acceptable. But readonly struct prevents field mutation.
   - Option B: Box the count in a single-element array: int[] _countHolder = new int[1]. This is one small allocation at construction time (acceptable -- it is construction, not parse path).
   - Option C: Use an unsafe approach.

   Go with Option B (int[1] for count tracking). The single int[] allocation happens at ErrorCollector construction, which is outside the per-parse zero-allocation path (the collector is constructed once per schema config, or once per parse call setup). This is the pragmatic approach that keeps readonly struct semantics clean.

4. Internal constructor: ErrorCollector(int capacity = 64) -- rents ValidationError[] from ArrayPool, creates int[1] for count
5. Public method: void Add(ValidationError error)
   - If count < capacity - 1: store error, increment count
   - If count == capacity - 1: replace with TooManyErrors sentinel (using ValidationErrorMessages.Get), set count = capacity
   - If count >= capacity: silently drop
6. Public properties: int Count, bool HasErrors (Count > 0)
7. Public indexer: this[int index] -- returns _errors[index] for valid range
8. Public method: Enumerator GetEnumerator() -- custom struct enumerator over _errors[0..Count]
9. Dispose(): return _errors to ArrayPool (null-check first)
10. XML doc comments on all public members

Per user decision: max error count configurable per schema (default 64), sentinel overflow at last slot.
